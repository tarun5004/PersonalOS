# Code of Conduct

## Our Pledge

We want Personal OS to be a respectful, welcoming, and useful open-source project.

Contributors are expected to communicate with patience, assume good intent, and focus feedback on the work rather than the person.

## Expected Behavior

- Be respectful and constructive.
- Keep discussions relevant to the project.
- Give actionable feedback.
- Welcome beginners and help them understand the docs-first workflow.
- Respect maintainers' phase boundaries and review decisions.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publicly sharing private information without consent.
- Repeatedly pushing out-of-scope changes after maintainers have declined them.
- Publishing suspected security vulnerabilities before maintainer review.

## Enforcement

Maintainers may moderate, edit, or remove comments, issues, pull requests, or other contributions that violate this code of conduct.

Maintainers may temporarily or permanently restrict participation when behavior harms the project or community.

## Reporting

Report conduct concerns privately to project maintainers through the repository's available private contact channel.

Security vulnerabilities should follow the private reporting process in `SECURITY.md`, not public issues.
