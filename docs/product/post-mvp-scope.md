# Post-MVP Scope

Status: Approved for V1 planning  
Source of truth: Master Prompt V4

## 1. Purpose

This document captures ideas that may be valuable after V1. These items are intentionally out of scope for the MVP unless the project owner approves a future phase.

## 2. Candidate Post-MVP Features

### Authentication and Accounts

- Password reset
- OAuth login
- Magic links
- Two-factor authentication
- Team accounts
- Admin panel
- Advanced roles and permissions

### Tasks

- Kanban board
- Drag-and-drop task ordering
- Subtasks
- Recurring tasks
- Task templates
- File attachments
- Team assignment
- Archive or trash recovery

### Habits

- Habit reminders
- Categories
- Templates
- Sharing
- Negative habits
- Advanced frequency rules
- Local timezone-based streaks

### Dashboard

- User-customizable dashboard layout
- Drag-and-drop widgets
- Saved dashboard presets
- Module marketplace or plugin registry

### Analytics

- Yearly analytics
- Exportable reports
- Predictive insights
- Advanced custom score weights
- Goal-based analytics

### Platform

- Mobile app
- Real-time collaboration
- Plugin marketplace
- AI features
- Advanced self-hosting templates

## 3. Post-MVP Decision Rule

A post-MVP feature should only move into planned scope when it has:

- A clear user problem.
- A documented product requirement.
- Architecture impact reviewed.
- Security and privacy implications reviewed.
- Acceptance criteria defined.
- A small implementation phase that can be reviewed independently.
