# Roadmap

Status: Approved for V1 planning  
Source of truth: Master Prompt V4

## 1. Roadmap Principle

Personal OS will be built phase by phase. Each phase will be small, reviewable, testable, and production-minded. Work will stop after each phase until the developer approves the next phase.

## 2. Documentation Foundation

### Phase 0-A: Product and UX Docs

This phase will define product requirements, personas, user stories, MVP scope, post-MVP scope, roadmap, information architecture, user flows, dashboard wireframe, habit tracker UX, and theme system UX.

### Phase 0-B: Engineering Docs

This phase will define engineering requirements, database design, API design, folder structure, component architecture, state management, theme architecture, security, and deployment documentation.

### Phase 0-C: Open Source Docs, ADRs, and Review Docs

This phase will define contributor docs, git workflow, coding standards, feature proposal process, pull request workflow, architecture decision records, technical risks, scalability notes, and final architecture review.

## 3. Build Phases

### Phase 1: Repository Setup

The repository will receive required root files, environment examples, basic scripts, and a Docker Compose stub.

### Phase 2: Frontend App Setup

The React and Vite frontend will be initialized with routing, basic structure, and temporary mock data where allowed.

### Phase 3: Backend App Setup

The Node and Express backend will be initialized with app structure, environment validation, middleware foundations, and health check.

### Phase 4: Theme System Foundation

The theme token system, light/dark themes, and theme context foundation will be added.

### Phases 5-7: Authentication and App Shell

The backend auth flow, frontend auth screens, protected routes, and app layout will be implemented.

### Phases 8-11: Tasks and Habits

Task and habit backend and frontend workflows will be implemented with validation, state handling, and tests.

### Phases 12-15: Dashboard and Analytics

Dashboard summary and analytics backend and frontend features will be implemented.

### Phases 16-18: Integration, Security, and UI Polish

The app will receive integration testing, security review, and UI polish.

### Phases 19-20: Documentation Polish and Release Prep

The project will be prepared for its first open-source release.

## 4. Release Target

The first open-source release will be version `0.1.0` after Phase 20.
